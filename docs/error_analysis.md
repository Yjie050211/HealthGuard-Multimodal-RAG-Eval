# Error Analysis

## Case 1: 正常休息被误判为长时间静止风险

- 输入：老人坐在沙发上长时间看电视。
- 可能错误：模型直接判断为中风险异常。
- 原因：缺少时间段、历史行为规律和主动确认信息。
- 改进：引入语音确认、历史活动规律和场景上下文。

## Case 2: 遮挡导致跌倒误判

- 输入：摄像头被家具遮挡，只能看到下半身。
- 可能错误：模型过度判断为跌倒。
- 原因：视觉证据不足。
- 改进：输出 uncertain，并触发人工复核。

## Case 3: 弱光下模型输出过度自信

- 输入：夜间画面模糊，人体轮廓不完整。
- 可能错误：模型给出确定风险等级。
- 原因：模型没有充分表达不确定性。
- 改进：在 prompt 中加入视觉质量约束，并在知识库中加入弱光处理规则。

## Case 4: RAG 检索到错误知识片段

- 输入：夜间异常徘徊。
- 可能错误：检索到跌倒处理建议。
- 原因：关键词重叠但语义不匹配。
- 改进：从 BM25 扩展到 embedding 检索或图结构检索。

## Case 5: 风险等级不稳定

- 输入：老人疑似摔倒后自行站起。
- 可能错误：有时输出 high，有时输出 medium。
- 原因：模型对短时跌倒与严重跌倒的边界不稳定。
- 改进：补充风险分级规则，并增加同类样本。
